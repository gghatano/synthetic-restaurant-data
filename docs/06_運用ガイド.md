# 06_運用ガイド

## 目的
生成したCSVが要件に合致しているかを、実行手順と確認観点で整理する。

## データ生成
### 基本
```bash
uv run fami-synth generate --start 2026-02-10 --end 2026-02-12 --seed 42 --out-dir data/output
```

### 標準運用ルール
- 出力先は `data/output` に統一する
- 期間指定は `start` 含む / `end` 含まない
- 再現性を担保するため、特別な理由がない限り seed は固定

### 上書きパラメータ例
```bash
uv run fami-synth generate \
  --start 2026-02-10 --end 2026-02-12 --seed 42 --out-dir data/output \
  --table-count 20 --visit-range 6,12 --order-range 1,2 --item-range 1,3 \
  --stay-range 30,90 --cook-range 240,900
```

### 日次増分
- 同一 `out-dir` で再実行すると、生成済み日付はスキップ
- 再生成する場合は `--force` を指定

## 可視化ダッシュボード
```bash
uv run fami-synth dashboard --data-dir data/output --host 127.0.0.1 --port 8000
```
ブラウザで `http://127.0.0.1:8000` を開く。

### ダッシュボードで確認する観点
- 来店数（店舗別）
- 時間帯分布
- 1来店あたり売上分布
- 1来店あたり注文数
- 1注文あたり商品数
- 割引利用数
- 税率適用分布
- ロイヤルティ提示率
- 季節/イベントメニュー比率

## 妥当性チェック（目視）
### 基本整合
- `visit -> order -> order_item -> receipt` の参照整合
- `receipt.subtotal_yen = order_item.line_subtotal_yen` の合計
- `receipt.total_yen = subtotal - discount + tax`
- `1 visit = 1 receipt`

### 営業時間
- `seated_at/ordered_at/served_at/paid_at` が `07:00-23:00` に収まる

## 自動テスト
```bash
uv run pytest -q
```

### テストで担保していること
- CSVヘッダ一致
- 日次増分（スキップ/force）
- 再現性
- 季節/イベントメニューの抽出
- 割引/税額/税率履歴の整合
- change_log の業務時間外記録
- ダッシュボードHTML生成
