# 00_進め方

## 方針
- 最小の仕組みを作り、Issue単位で段階的に機能追加する。
- まずは「日次増分」「CSV出力」「再現性」「1 visit = 1 receipt」を完成させる。
- マスタ（メニュー履歴/割引/税率履歴）と季節/イベントを追加済み。

## 推奨Issue分割（順番）
1. CLI + CSV出力の骨格（ヘッダ固定、1日生成）
2. 日次増分（生成済み日のスキップ、--force）
3. 再現性テスト（同一seedで一致）
4. メニュー（通常のみ）+ 価格 + 注文内容の妥当化
5. 割引（常時2本）
6. 税率履歴（tax_rate_history）+ receiptへの参照ID
7. 季節メニュー
8. イベントメニュー（±7日）
9. change_log（オフ時間に記録、翌07:00有効）
